# ES 評価機能 ユーザーフローガイド

## 📋 概要

この LINE Bot は、エントリーシート（ES）を AI で評価する機能を提供します。ES 本文を送信すると、10 観点でのスコアリングと企業別の適合度評価が返されます。

## 🔄 ユーザーフロー

### 1. 友だち追加時

1. **LINE Bot を友だち追加**
   - 自動的にイベントカードが表示されます
   - LINE ユーザー ID が自動的に取得・保存されます

### 2. ES 評価の基本フロー

#### パターン A: ES 本文のみを送信（企業指定なし）

```
ユーザー → ES本文をそのまま送信
         ↓
Bot → ES評価を実行（ベース評価）
     → 10観点スコアリング結果を返信
     → 全企業アドオン企業での適合度評価を返信
```

**例:**

```
ユーザー: 大学時代にサークル活動でリーダーを務めました。
         メンバー間の意見の対立を解決し、イベントを成功させました。
         この経験から、コミュニケーションの重要性を学びました。
```

#### パターン B: 企業指定ありで ES 本文を送信

```
ユーザー → 「企業: パナソニック」+ ES本文を送信
         ↓
Bot → 企業アドオンを適用してES評価を実行
     → パナソニック向けの評価結果を返信
```

**例:**

```
ユーザー: 企業: パナソニック

         大学時代にサークル活動でリーダーを務めました。
         メンバー間の意見の対立を解決し、イベントを成功させました。
         この経験から、コミュニケーションの重要性を学びました。
```

#### パターン C: 企業設定を先に保存してから ES を送信

```
ユーザー → 「企業: パナソニック」のみを送信
         ↓
Bot → 企業設定を保存
     → 「続けてES本文を送ってください」と返信
         ↓
ユーザー → ES本文を送信
         ↓
Bot → 保存された企業設定を適用してES評価を実行
```

**例:**

```
ユーザー: 企業: パナソニック
Bot: 企業設定を保存しました：パナソニック
     企業アドオン: パナソニック（適用）
     続けてES本文を送ってください。

ユーザー: 大学時代にサークル活動でリーダーを務めました...
```

## 📝 使い方詳細

### ES 本文の送信方法

1. **最小文字数**: 80 文字以上
2. **最大文字数**: 2000 文字まで（推奨）
3. **個人情報**: 伏せて OK（名前、大学名など）

### 企業アドオンの指定方法

**対応企業:**

- パナソニック
- ロート製薬
- 野村（證券/グループ）
- 旭化成
- NTT 西日本
- 三井住友海上あいおい生命
- KDDI
- 阪急阪神ホールディングス

**指定形式:**

- `企業: パナソニック`
- `会社: ロート製薬`
- `応募企業=KDDI`

### コマンド一覧

| コマンド   | 説明                         | 例         |
| ---------- | ---------------------------- | ---------- |
| `ヘルプ`   | 使い方を表示                 | `ヘルプ`   |
| `リセット` | 企業設定と続き状態をリセット | `リセット` |
| `続き`     | 前回の評価結果の続きを表示   | `続き`     |

## 📊 評価結果の内容

評価結果には以下の情報が含まれます：

1. **A) ES 要約**

   - STAR 形式（Situation/Task/Action/Result/Learning）

2. **B) 10 観点スコアカード**

   - (1) 課題発見・問題解決力
   - (2) チャレンジ精神・向上心
   - (3) 主体性・行動力
   - (4) やり抜く力（粘り強さ）
   - (5) 協働性・チームワーク
   - (6) コミュニケーション・周囲巻き込み力（EQ/影響力含む）
   - (7) 誠実性・倫理観（Integrity）
   - (8) 学習意欲・自己成長力（Learning Agility）
   - (9) 適応力・柔軟性
   - (10) 顧客志向・社会貢献マインド（利他・ステークホルダー配慮）
   - 各観点で 0〜5 点（または NA）と確信度（高/中/低/NA）

3. **C) 企業別「らしさ」適応メモ**

   - 全企業アドオン企業での適合度評価
   - 企業重み付きスコア（品質/保守/重みカバレッジ）

4. **D) リスク/懸念**

   - [CRITICAL]: 重大な懸念
   - [CHECK]: 要確認事項

5. **E) スコア（機械可読）**
   - SCORES: 各観点のスコア
   - CONF: 各観点の確信度
   - SIGNALS: Fit シグナル（0/1/NA）
   - BASE: ベーススコア
   - COMPANY: 企業別スコア
   - TOP3: 上位 3 企業
   - IMPROVE_TOP3: 改善優先 3 観点
   - DECISION: 判定（通過推奨/保留/見送り）

## ⚠️ 注意事項

1. **文字数制限**

   - 80 文字未満の場合は評価されません
   - 2000 文字を超える場合は評価が不安定になる可能性があります

2. **続きメッセージ**

   - 評価結果が長い場合、複数のメッセージに分割されます
   - 「続き」コマンドで残りを表示できます

3. **企業設定の保持**
   - 一度企業を設定すると、次回の ES 送信時にも適用されます
   - リセットする場合は「リセット」コマンドを使用してください

## 🔧 トラブルシューティング

### ES が評価されない場合

1. **文字数確認**: 80 文字以上あるか確認
2. **企業名の誤字**: 企業名が正しく入力されているか確認
3. **「ヘルプ」コマンド**: 使い方を再確認

### 評価結果が途中で切れる場合

- 「続き」コマンドで残りを表示できます

### 企業設定を変更したい場合

- 「リセット」コマンドでリセット後、新しい企業名を指定してください
